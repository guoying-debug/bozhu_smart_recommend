设计带意图识别与RAG的Coze工作流编排

下面给你一套可以直接在 Coze 里照着搭的“完整工作流设计”（含：输入 → 意图识别 → 闲聊分流 → 选题链路 → RAG → 工具调用 → 输出），并把每个节点的 入参/出参/提示词模板/路由条件/失败兜底 都写清楚。

## 0. 全局约定（工作流变量）
建议在 Coze 工作流里统一用这些变量贯穿：

- user_query ：用户原话
- intent ：意图分类结果
- profile ：用户画像/创作约束（对象）
  - domain （赛道/领域）
  - goal （涨粉/变现/稳定流量）
  - format （口播/测评/盘点/教程/剧情）
  - level （新手/进阶）
  - constraints （时长/是否出镜/更新频率等）
- topics_raw ：getTopics 原始返回
- topic_options ：筛选后可选方向列表（给用户选择）
- selected_topic ：用户选中的方向（含 cluster_id、方向名、热词、参考标题）
- audience_profile ：人群定位结果（结构化）
- candidate_titles ：候选标题数组
- scored_titles ：候选标题 + 分档预测 + 概率
- final_plan ：最终输出（结构化）
## 1) 节点：输入（Input）
- 输入：用户对话 user_query
- 输出：写入 user_query
## 2) 节点：意图识别（Intent Router / LLM 分类）
### 2.1 分类标签（建议 6 类）
- CHITCHAT ：你好/谢谢/在吗/表情包/无任务
- CREATOR_GUIDE ：如何当博主/起号/运营/涨粉方法论
- TOPIC_DISCOVERY ：选题方向/热门话题/我该做什么内容
- TITLE_EVAL ：给定标题想评估播放/分档/优化
- OTHER ：无法归类
### 2.2 提示词（节点模板）
输出必须是严格 JSON：

```
你是意图分类器。根据用户输入，把意图归类为：
CHITCHAT / CREATOR_GUIDE / TOPIC_DISCOVERY / TITLE_EVAL /  OTHER
只输出JSON，不要输出其它文字。

用户输入：{{user_query}}

输出JSON格式：
{"intent":"...", "confidence":0-1, "slots":{"domain":"","goal":"","format":"",
"level":""}}
```
- 输出保存：
  - intent
  - profile.domain/profile.goal/profile.format/profile.level （能抽就抽，抽不到留空）
### 2.3 路由条件
- intent == CHITCHAT → 走闲聊分支
- intent == CREATOR_GUIDE → 走“方法论 + RAG”分支
- intent == TOPIC_DISCOVERY → 走“选题链路”分支
- intent == TITLE_EVAL → 走“标题评估链路”
- intent == PLUGIN_DEBUG → 走“排障链路”
- else → 走兜底澄清（但不要问太多，一次问 1–2 个关键问题）
## 3) 分支 A：闲聊机器人（CHITCHAT）
- 不调用工具
- 直接回复友好闲聊 + 引导一句：
  - “你想做选题推荐/人群定位/标题预测哪个？给我一句话描述你的领域就行。”
## 4) 分支 B：博主方法论（CREATOR_GUIDE）+ RAG
### 4.1 RAG 资料库设计（建议两个知识库）
- KB_运营方法论 ：起号、内容结构、选题方法、标题与封面、更新节奏、数据复盘
- KB_你的项目说明/能力边界 ：你现有项目能做什么（聚类、相似标题、分档预测）、注意事项（模型只预测“相对潜力”等）
资料来源你可以直接用你项目里已有内容作为第一批 KB，例如：

- 「博主智策」全项目设计与实施方案（MCP+Function Call+Coze）.md
### 4.2 RAG 节点（Retrieve）
- Query： user_query + profile.domain + profile.goal
- TopK：3–6
- 输出： rag_context
### 4.3 大模型节点（生成回答）
- 输入： user_query + rag_context
- 输出：给出“可执行清单”，结构建议：
  - 目标拆解（涨粉/变现）
  - 7天执行计划（每天发什么）
  - 复盘指标（播放、完播、互动、转粉）
## 5) 分支 C：选题链路（TOPIC_DISCOVERY）= 你要的主流程
### 5.1 节点：补全关键信息（轻问 1–2 个问题）
如果 profile.domain/goal/format 缺失严重，只问最关键 1–2 个：

- “你主要想做哪个领域？”
- “你做视频主要目标是涨粉还是变现？”
  不要问一堆。
### 5.2 工具节点：getTopics
- 调用： GET /api/topics
- 输出： topics_raw
### 5.3 RAG 节点：结合方法论筛选方向（可选但很有用）
- Query： profile + "如何筛选赛道/选题方向/人群定位"
- 输出： rag_context_topics
### 5.4 大模型节点：筛选 + 命名方向（输出给用户选择）
输入： topics_raw + profile + rag_context_topics

输出必须结构化，便于后续“选择节点”解析：

```
{
  "topic_options": [
    {
      "option_id": 1,
      "cluster_id": 3,
      "name": "AI工具效率流",
      "why_fit": "适合新手，教程导向，供给强需求",
      "top_terms": ["AI","工具","教程"],
      "reference_titles": ["...","..."],
      "risk": "同质化高，需要细分人群"
    }
  ]
}
```
保存为： topic_options

### 5.5 选择节点：用户选方向
- 让用户输入 1/2/3（或直接点选）
- 保存： selected_topic
### 5.6 大模型节点：人群画像（Audience Profiling）
输入： selected_topic + profile + rag_context_topics

输出结构：

```
{
  "audience_profile": {
    "primary_audience": "xx人群",
    "pain_points": ["...","..."],
    "triggers": ["标题表达偏好...","内容结构偏好..."],
    "content_promises": ["你能帮他解决什么"],
    "tone": "理性/情绪/陪伴/干货"
  }
}
```
保存： audience_profile

### 5.7 大模型节点：生成候选标题（10条）
输入： selected_topic + audience_profile + profile.format

输出结构：

```
{"candidate_titles":[{"title":"...","category":"未知"}]}
```
保存： candidate_titles

### 5.8 循环节点：对候选标题做分档预测（predictBucket）
- 循环 10 次
- 每次调用： POST /api/predict_bucket
  - Body： {"title": item.title, "category": item.category, "author_id": profile.author_id, "publish_time": profile.publish_time}
- 汇总输出： scored_titles
  - 结构包含：
    - title
    - predicted_bucket_name
    - bucket_probabilities
### 5.9 RAG 节点：把输出变成“可落地选题方案”（增强可执行性）
- Query： "选题落地模板/脚本大纲/封面标题技巧/避坑"
- 输出： rag_context_plan
### 5.10 输出节点：最终交付（Final Plan）
把 scored_titles 按优先级排序输出（建议优先：50-200万、200万+，同时保留2条稳健的0-10万做“新手练手”）。

输出格式建议：

- 方向总结（1段）
- 人群定位（3–5条）
- 选题清单表格（Top 10）
  - 标题 | 目标人群 | 预期分档 | 概率Top2 | 脚本大纲（3点） | 封面文案
## 6) 分支 D：标题评估链路（TITLE_EVAL）
当用户直接丢一个标题：

- 先用 predictBucket 给分档 + 概率
- 再用 titleAdvice 给相似爆款 + 关键词
- 大模型综合生成：3个可改进版本标题 + 为什么更好（但别太长）
