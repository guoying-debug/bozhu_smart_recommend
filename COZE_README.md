# Coze 工具说明文档：博主智策 (Bilibili Creator Assistant)

本文档旨在为 Coze 插件配置和 Prompt 编写提供详细说明，帮助你更好地理解每个 API 工具的输入输出参数及其业务含义。

---

## 工具 1: `getTopics`

### 工具描述
**获取话题簇摘要（K-Means）**。该工具不接受任何输入参数，直接返回当前 B 站最热门的“话题聚类”结果。系统后台会自动分析最近抓取的视频标题，通过 K-Means 算法将其归类，并提取每个类别的核心关键词和代表性视频。

### 适用场景
当用户询问“最近 B 站流行什么？”、“有哪些热门选题？”或“给我一些创作灵感”时，调用此工具。

### 输出参数详解 (Response Parameters)

该接口返回一个数组 (Array)，数组中的每一项代表一个**话题簇 (Topic Cluster)**。

| 参数名称 (Key) | 类型 (Type) | 业务含义 (Description) | 示例值 (Example) |
| :--- | :--- | :--- | :--- |
| **cluster_id** | Integer | **话题编号**。用于区分不同的话题类别。 | `3` |
| **top_terms** | Array<String> | **核心关键词**。该话题下出现频率最高、TF-IDF 权重最高的词汇，反映了该话题的主题。Bot 应根据这些词总结话题名称。 | `["测评", "开箱", "上手", "体验"]` |
| **total_videos** | Integer | **视频总数**。该话题簇中包含的热门视频数量，反映话题的热度广度。 | `156` |
| **total_views** | Integer | **总播放量**。该话题下所有视频播放量之和，反映话题的流量池规模。 | `85000000` |
| **average_views** | Number | **平均播放量**。`total_views / total_videos`，反映该话题的平均流量水平，用于评估“容易火”的程度。 | `544871.8` |
| **top_videos** | Array<Object> | **代表性视频**。该话题下播放量最高的 Top 5 视频，供用户参考具体内容。 | (见下表) |

#### `top_videos` 内部结构

| 子参数名称 | 类型 | 含义 |
| :--- | :--- | :--- |
| **title** | String | 视频标题。 |
| **view_count** | Integer | 视频播放量。 |

## 工具 2: `getAnalysisSummary`

### 工具描述
**获取聚类分析摘要**。该工具返回数据分析的宏观统计信息，包括 DBSCAN 和 K-Means 算法的聚类结果概览。它可以帮助用户了解当前数据池的整体结构，例如有多少视频被有效归类，有多少是“噪声”（不属于任何热门话题的独特视频）。

### 适用场景
当用户询问“目前的分析数据概况如何？”、“有多少视频参与了分析？”或“数据质量怎么样？”时，调用此工具。

### 输入参数 (Request)
无。

### 输出参数详解 (Response Parameters)

| 参数名称 (Key) | 类型 (Type) | 业务含义 (Description) | 示例值 (Example) |
| :--- | :--- | :--- | :--- |
| **total_videos** | Integer | **参与分析的视频总数**。 | `5000` |
| **kmeans_clusters** | Object | **K-Means 聚类分布**。Key 是簇ID，Value 是该簇包含的视频数量。 | `{"0": 120, "1": 85...}` |
| **dbscan_noise_ratio** | Number | **DBSCAN 噪声比例**。无法被归类的视频占比。比例越低，说明热点越集中。 | `0.15` |
| **dbscan_clusters_count** | Integer | **DBSCAN 发现的高密度簇数量**。 | `12` |

## 工具 3: `predictView`

### 工具描述
**预测播放量（回归）并返回分档结果**。该工具基于机器学习模型（Ridge Regression），根据用户输入的视频标题、分区、作者等信息，预测视频的潜在播放量数值，并对其进行分档（如“10-50万”）。同时，它还会返回影响预测结果的关键因子分析。

### 适用场景
当用户询问“这个标题能有多少播放量？”、“预测一下我的视频热度”时，调用此工具。

### 输入参数详解 (Request Parameters)

| 参数名称 | 必填 | 类型 | 示例值 | 业务含义 |
| :--- | :--- | :--- | :--- | :--- |
| **title** | 是 | String | `新手入门：3天做出爆款视频？` | **视频标题**。预测的核心依据。 |
| **category** | 否 | String | `知识` | **分区/类目**。不同分区的流量基数不同，默认为“未知”。 |
| **author_id** | 否 | Integer/String | `123456` | **作者ID**。如有，模型会参考该作者的历史平均播放量；无则按“冷启动”处理。 |
| **publish_time** | 否 | Integer | `1700000000` | **发布时间**。用于分析发布时段对流量的影响。 |

### 输出参数详解 (Response Parameters)

| 参数名称 (Key) | 类型 (Type) | 业务含义 (Description) | 示例值 (Example) |
| :--- | :--- | :--- | :--- |
| **predicted_view** | Number | **预测播放量**。模型给出的具体预估数值。 | `125000.5` |
| **predicted_bucket** | String | **预测分档**。将播放量划分为 4 档：`0-10万`、`10-50万`、`50-200万`、`200万+`。 | `10-50万` |
| **bucket_id** | Integer | **分档ID**。对应上述分档的数字编号 (0-3)。 | `1` |
| **explanations** | Array<Object> | **影响因子分析**。解释为什么预测结果是高或低。 | (见下表) |

#### `explanations` 内部结构

| 子参数名称 | 类型 | 含义 | 示例 |
| :--- | :--- | :--- | :--- |
| **feature** | String | **特征名称**。影响播放量的具体因素。 | `标题长度`、`疑问句式`、`分区定位` |
| **effect** | String | **影响方向**。`正向`（加分）或 `负向`（减分）。 | `正向` |
| **reason** | String | **具体原因**。对该特征影响的自然语言解释。 | `标题包含丰富信息`、`疑问句引发好奇心` |

## 工具 4: `predictBucket`

### 工具描述
**预测播放量分档**。与 `predictView` 不同，该工具专注于**分类任务**（Logistic Regression）。它将视频的潜在播放量划分为 4 个固定的等级区间（0-10万 / 10-50万 / 50-200万 / 200万+），并给出属于每个等级的概率。

### 适用场景
当用户不需要具体的播放数字，只想知道视频处于什么“段位”（如：是小热还是爆款？）时，或者当回归预测不准时，用分类模型作为辅助参考。

### 输入参数详解 (Request Parameters)
同 `predictView`。

### 输出参数详解 (Response Parameters)

| 参数名称 (Key) | 类型 (Type) | 业务含义 (Description) | 示例值 (Example) |
| :--- | :--- | :--- | :--- |
| **predicted_bucket** | String | **预测分档名称**。最终判定该视频属于哪个流量池。 | `10-50万` |
| **bucket_id** | Integer | **分档ID**。对应上述分档的数字编号 (0-3)。 | `1` |
| **probabilities** | Object | **各档位概率**。模型认为该视频属于各个档位的可能性，Key 是档位名称，Value 是 0-1 的概率值。 | `{"0-10万": 0.2, "10-50万": 0.7, ...}` |

## 工具 4: `titleAdvice`

### 工具描述
**标题诊断与建议（AI 增强版）**。这是本插件最核心的工具。它不仅仅是一个简单的预测工具，而是结合了**预测模型（定量分析）**和**大语言模型（定性分析）**的智能体。它会先在数据库中检索相似的爆款视频，计算标题的特征得分，然后综合这些信息生成一份深度的诊断报告和具体的改写建议。

### 适用场景
当用户询问“这个标题好不好？”、“如何优化我的标题？”、“帮我改个爆款标题”时，调用此工具。

### 输入参数详解 (Request Parameters)

| 参数名称 | 必填 | 类型 | 示例值 | 业务含义 |
| :--- | :--- | :--- | :--- | :--- |
| **title** | 是 | String | `新手入门：3天做出爆款视频？` | **待诊断的视频标题**。这是核心输入。 |
| **category** | 否 | String | `知识` | **视频分区**。如果不填，默认为“未知”。不同分区的评分标准不同（例如知识区看重干货，生活区看重共鸣）。 |
| **author_id** | 否 | Integer/String | `123456` | **作者ID**。用于个性化分析（如历史平均播放量），目前版本主要用于记录日志。 |
| **publish_time** | 否 | Integer | `1700000000` | **发布时间戳**。用于分析发布时段的影响（如黄金时间 vs 深夜）。 |

### 输出参数详解 (Response Parameters)

| 参数名称 (Key) | 类型 (Type) | 业务含义 (Description) | 示例值 (Example) |
| :--- | :--- | :--- | :--- |
| **diagnosis** | String | **智能体诊断报告**。这是 LLM 生成的一段详细文本，分析了原标题的优缺点（如：“标题使用了疑问句增加了好奇心，但缺乏情绪词，略显平淡”）。Bot 应直接展示给用户。 | `标题结构清晰，但吸引力不足...` |
| **advice_list** | Array<String> | **优化建议列表**。系统生成的 3 个具体的改写方案，Bot 可以让用户从中选择。 | `["揭秘！3天做出爆款的秘密", ...]` |
| **summary** | String | **总结摘要**。简要说明找到了多少相似视频，以及整体评价。 | `为您找到 5 个相似热门标题...` |
| **similar_titles** | Array<Object> | **相似爆款参考**。数据库中已存在的、和输入标题相似的高播放视频，供用户寻找灵感。 | (见下表) |

#### `similar_titles` 内部结构

| 子参数名称 | 类型 | 含义 |
| :--- | :--- | :--- |
| **title** | String | 相似视频的标题。 |
| **view_count** | Integer | 该视频的实际播放量（证明它是爆款）。 |
| **similarity** | Number | 相似度得分 (0-1)。越接近 1 表示越相似。 |
