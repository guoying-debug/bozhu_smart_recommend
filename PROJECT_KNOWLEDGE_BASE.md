# 博主智策：项目说明与能力边界知识库

本文档旨在为 Coze 智能体（Bot）提供核心知识库，帮助 Bot 准确理解项目功能、数据来源及能力边界，从而更专业地回答用户提问。

---

## 1. 项目核心定位

**项目名称**：博主智策 (Bilibili Creator Assistant)
**核心价值**：面向 B 站自媒体博主，提供基于数据驱动的“选题挖掘”与“内容优化”辅助工具。
**技术原理**：
1.  **数据采集**：通过爬虫获取 B 站近期热门视频数据（标题、播放量、分区等）。
2.  **聚类分析**：使用 BERT + K-Means/DBSCAN 算法对热门标题进行语义聚类，发现当前流行的话题簇。
3.  **预测模型**：基于随机森林/岭回归模型，根据标题特征（长度、情绪词、符号等）和分区，预测视频的潜在播放量级。
4.  **智能诊断**：结合 LLM（大语言模型）与特征重要性分析，对用户输入的标题进行深度诊断并提供优化建议。

---

## 2. 核心功能与 API 能力

### 2.1 热门话题挖掘 (`/api/topics`)
*   **功能**：展示当前 B 站最热的“话题簇”。
*   **数据逻辑**：将数千条热门视频标题转化为向量，聚类成若干个话题（如“数码测评”、“游戏实况”、“美食探店”）。
*   **返回信息**：
    *   `cluster_id`: 话题编号。
    *   `top_terms`: 该话题的核心关键词（如：测评、开箱、不仅是...）。
    *   `average_views`: 该话题下视频的平均播放量（反映流量池大小）。
    *   `representative_titles`: 代表性视频标题。

### 2.2 播放量预测 (`/api/predict_view`)
*   **功能**：用户输入标题和分区，系统预估其播放量。
*   **能力边界**：
    *   **仅供参考**：预测基于历史热门数据的统计规律，**不是算命**。它反映的是“具有此类特征的标题在当前环境下**通常**的表现”，而非该视频实际发布的绝对结果。
    *   **影响因子**：模型会分析“标题长度”、“疑问句”、“特殊符号（【】）”、“分区定位”等特征对播放量的正/负向影响。
    *   **冷启动假设**：默认假设作者处于“中等水平”的粉丝基础，无法精准预测头部大 V 或纯素人的极端情况。

### 2.3 标题优化建议 (`/api/title_advice`)
*   **功能**：AI 智能体对用户标题进行“会诊”。
*   **工作流**：
    1.  **检索 (RAG)**：在数据库中寻找与用户输入相似的已爆款视频标题。
    2.  **特征分析**：计算标题的各项指标（如是否包含情绪词）。
    3.  **LLM 诊断**：结合全局特征重要性（哪些词更容易火），生成针对性的修改建议。
*   **输出**：
    *   `diagnosis`: 深度诊断报告（如：“你的标题太像说明书了，缺乏情绪价值”）。
    *   `advice_list`: 3 个具体的修改建议（如：“建议改为：...？”）。
    *   `similar_titles`: 相似的爆款标题供参考。

## 3. 能力边界与常见问题 (FAQ)

### Q1: 预测的播放量准吗？
**答**：**不保证绝对准确。** 
*   **局限性**：播放量受封面图、视频内容质量、发布时间、粉丝粘性等多种因素影响，标题只是其中之一。
*   **价值**：预测值的意义在于**横向对比**。如果优化后的标题预测值比原标题高，说明从统计学上看，新标题具有更高的点击潜力。

### Q2: 为什么我看不到“情绪词”的分析了？
**答**：项目已升级。早期的“情绪词”词库分析因准确度不足已被移除。现在我们使用更高级的 **“分区定位”影响因子**（如知识区 vs 生活区的流量差异）和 LLM 的语义理解来替代简单的词库匹配。

### Q3: 为什么建议我换分区？
**答**：不同分区的流量天花板不同。模型可能会发现你的标题风格在“生活区”比在“知识区”更容易获得高播放（例如标题很标题党，但内容偏娱乐），因此会给出分区建议。

### Q4: 你们的数据是实时的吗？
**答**：是**准实时**的。系统每天凌晨会自动运行爬虫和聚类脚本，更新最近 24 小时内的热门趋势。但 API 返回的是上一次批量处理的结果，不是秒级更新。

### Q5: Docker 部署需要什么配置？
**答**：
*   **内存**：建议至少 4GB（加载 BERT 模型和数据需要）。
*   **网络**：如果在国内部署，需配置 Docker 镜像加速（如华为云/阿里云镜像），否则拉取基础镜像会失败。
*   **端口**：默认占用 5000 (Web), 3000 (Grafana), 9090 (Prometheus), 3307 (MySQL 映射)。

## 4. 专有名词解释

*   **Bucket (分档)**：将播放量划分为 4 个等级：
    *   0: 0-10万 (普通)
    *   1: 10-50万 (小热)
    *   2: 50-200万 (大热)
    *   3: 200万+ (爆款)
*   **Silhouette Score (轮廓系数)**：用于评估聚类效果好坏的指标（越接近 1 越好），项目用它自动决定聚成多少类。
*   **Training-Serving Skew**：训练服务偏差。本项目通过统一 `feature_utils.py` 代码库，完美解决了这个问题，确保训练和预测逻辑一致。
